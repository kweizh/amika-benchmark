# Initialize a Jujutsu Repository and Configure User Identity You are a developer who has recently switched from Git to Jujutsu (jj). You have an existing project directory at `/home/user/myproject` that you want to bring under jj version control. Your goal is to initialise a jj repository in that directory and set your user identity in the global jj configuration so that commits are attributed to you. ## Background Jujutsu (jj) is a modern version-control system with first-class support for Git backends. Unlike Git, jj separates the concept of a "change" (an intention to modify the codebase) from a "commit" (a snapshot of the repo at a point in time). Every working-copy state is automatically snapshotted, so there is no explicit staging area. When you first use jj you must configure two pieces of identity information that will be embedded in every commit you author: - **user.name** â€“ your display name - **user.email** â€“ your e-mail address These values live in the user-level (global) configuration file, not the repository-level file. On Linux the global config is stored at `~/.config/jj/config.toml` (or the path returned by `jj config path --user`). ## Starting Environment - You are logged in as `user` with home directory `/home/user`. - The directory `/home/user/myproject` already exists and contains a single file: `README.md` with the content `# My Project`. - `jj` (version 0.38.0) is installed and available on `PATH` as `/usr/local/bin/jj`. - No jj repository has been initialised yet anywhere under `/home/user`. - No jj user-level configuration exists yet (i.e. `~/.config/jj/config.toml` does not exist). ## Your Tasks ### Task 1 â€” Initialise a jj repository Initialise a jj repository **with a Git backend** inside `/home/user/myproject`.
