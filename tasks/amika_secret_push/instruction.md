# Push an API key to Amika Cloud Vault

## Background
You have the `amika` CLI installed and authenticated. You need to store a Claude API key securely in the Amika cloud vault so it can be injected into remote sandboxes.

## Requirements
- Push a mock Claude API key `sk-ant-api03-mock-key-123` to the Amika secret store as type `api_key`.
- The secret must be named `mock-key`.

## Constraints
- Project path: /home/user/amika-project
- Use the `amika` CLI to push the secret.

## Integrations
- None
