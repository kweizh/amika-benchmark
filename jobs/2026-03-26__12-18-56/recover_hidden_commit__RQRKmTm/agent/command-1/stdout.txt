# Recovering a Hidden (Abandoned) Commit in Jujutsu ## Background As an OS engineer comparing `jj` to Git, you are exploring how Jujutsu handles abandoned commits that are no longer visible in the default log. In Jujutsu, the analogous concept is a **hidden** commit. When you run `jj abandon <rev>`, the commit is not deleted — it is hidden from the default log view but still stored. Unlike Git's reflog, Jujutsu provides a first-class **operation log** (`jj op log`) that records every mutation. Using the operation log, you can rediscover commits that were abandoned in earlier operations. ## Starting Environment You have a jj repository at `/home/user/kernel-patch`.
