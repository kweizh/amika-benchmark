# Create and Move Bookmarks in a jj Repository ## Context You are a developer who has recently migrated from Git to Jujutsu (`jj`). You are working on a small web application stored in a `jj` repository at `/home/user/webapp`. In Jujutsu, **bookmarks** serve the same role as Git branches — they are named pointers to specific commits. Unlike Git, Jujutsu does not have an "active" branch concept; bookmarks are simply labels that you move around manually or via specific commands. The repository currently has the following commit history (oldest to newest): 1. **`chore: init webapp`** — the root commit, adds `README.md` and `app/__init__.py` 2. **`feat: add homepage route`** — introduces `app/routes.py` with a basic homepage handler 3. **`feat: add contact page`** — introduces `app/contact.py` with a contact page handler Commits 2 and 3 are stacked linearly above the root. The working copy (`@`) is currently an empty commit sitting on top of commit 3 (i.e., `@-` is `feat: add contact page`). In jj, commit 3 is the **parent of the working copy**, which can be referenced with the revset `@-`. Use the `jj bookmark create` command with the `-r` option to specify the target revision: ``` jj bookmark create <bookmark-name> -r <revision> ``` The bookmark name must be exactly `feature/contact` (with a forward slash). You do not need to use `sudo`, and you do not need to modify any files — only bookmark management commands are required. After creating this bookmark, verify it exists by running `jj bookmark list`. You need to move it **forward** to also point to commit 3 (`feat: add contact page`, i.e., `@-`). In jj, moving an existing bookmark is done with `jj bookmark move`: ``` jj bookmark move <bookmark-name> --to <revision> ``` Moving a bookmark **forward** (to a descendant) does not require any special flags. The `--allow-backwards` flag is only needed when moving a bookmark to an ancestor or unrelated commit. Since commit 3 is a descendant of commit 2, you can move `dev` directly: ``` jj bookmark move dev --to @- ``` > **Note for Git migrants:** In Git, you would use `git branch -f dev <commit>` to move an existing branch.
