# Prune Stale WIP Revisions Using Revsets

## Background

You are an open-source maintainer working on a project called `mylib` hosted on GitHub. You use Jujutsu (`jj`) as your version control system over a Git-backed repository.

Over the past few days you have been experimenting locally and left behind several scratch/WIP (work-in-progress) revisions that were never promoted to a proper bookmark or given a meaningful description. You now want to clean up the repository by **abandoning** those stale WIP revisions before pushing your real work upstream.

The repository already has the following history set up for you:

```
 root()
  └── "Initial project scaffold"         ← bookmark: main
       └── "Add core library module"      ← bookmark: feature/core
            ├── "(no description set)"    ← WIP scratch commit A (no bookmark)
            └── "(no description set)"    ← WIP scratch commit B (no bookmark)
```

Both WIP commits are:
- Direct children of the `feature/core` bookmark commit
- Empty of any useful description (description is blank / empty string)
- Not pointed to by any local bookmark

Your task is to **abandon all mutable revisions that have an empty description AND are not ancestors of any local bookmark**, then verify the result.

## Starting Environment

- Working directory: `/home/user/mylib`
- The repository is already initialised as a `jj` git-backed repo
- User identity is already configured in jj config
- The commit history described above is already in place
- `jj` binary is available in `PATH`

## Your Task

Complete the following steps **inside `/home/user/mylib`**:

### Step 1 — Inspect the current log

Run `jj log` to see the current history and identify the WIP revisions:

```bash
cd /home/user/mylib
jj log
```

You should see at least four revisions: the root, `main`, `feature/core`, and two WIP commits that have no description and no bookmark.

### Step 2 — Abandon all WIP revisions using a revset

Use a single `jj abandon` command with a revset expression to abandon **all mutable revisions that have an empty description and are not ancestors of any local bookmark**.

The revset you need is:

```
mutable() & empty_description & ~::bookmarks()
```

where `empty_description` is expressed using the `description("")` function (which matches commits whose description is the empty string).

The exact command to run:

```bash
jj abandon 'mutable() & description("") & ~::bookmarks()'
```

This revset reads as:
- `mutable()` — only commits that are not immutable (i.e., not protected remote/tag heads)
- `description("")` — only commits whose description is the empty string
- `~::bookmarks()` — exclude commits that are ancestors-of (or equal to) any local bookmark target

**Important**: The `~::bookmarks()` part uses the `::` range operator to compute the set of all ancestors of bookmark targets (inclusive), and then negates it with `~`. This ensures you only abandon commits that are genuinely not reachable from any named bookmark.

### Step 3 — Verify the result

After abandoning, confirm the cleanup was successful:

```bash
jj log
```

You should now see **only** the revisions that have bookmarks (`main`, `feature/core`) and the working-copy commit. The two WIP scratch commits must no longer appear in the visible history.

Also verify with a revset query:

```bash
jj log -r 'mutable() & description("") & ~::bookmarks()'
```

This should output **nothing** (or an empty result), confirming all matching WIP revisions have been abandoned.

## Expected Final State

- The two WIP revisions (empty description, no bookmark) are **abandoned** (no longer visible)
- The `main` bookmark still points to the "Initial project scaffold" commit
- The `feature/core` bookmark still points to the "Add core library module" commit
- `jj log -r 'mutable() & description("") & ~::bookmarks()'` returns an empty result
- The file `/home/user/mylib/README.md` still exists (it was part of the scaffold)
- The file `/home/user/mylib/src/lib.rs` still exists (it was part of the core module)

## Verification Conditions

Automated tests will check:
1. `jj log -r 'mutable() & description("") & ~::bookmarks()'` in `/home/user/mylib` returns an empty result (exit 0 but no commits shown, or the message indicating no revisions)
2. `jj bookmark list` shows both `main` and `feature/core` bookmarks still present
3. The file `/home/user/mylib/README.md` exists
4. The file `/home/user/mylib/src/lib.rs` exists
