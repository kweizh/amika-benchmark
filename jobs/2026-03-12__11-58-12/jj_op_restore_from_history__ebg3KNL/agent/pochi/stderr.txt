## Key Takeaway: jj op log vs git reflog

| Aspect | Git | jj |
|--------|-----|-----|
| History viewer | `git reflog` (SHA-based) | `jj op log` (operation-based with descriptions) |
| Preview past state | `git show <sha>` | `jj --at-op=<op-id> log` |
| Restore | `git reset --hard <sha>` (destructive) | `jj op restore <op-id>` (non-destructive, adds new operation) |
| Granularity | Per-commit | Per-operation (squash, rebase, describe, etc.) |

The `jj op restore` approach is safer than `git reset --hard` because it never deletes history â€” it always appends a new restore operation on top, making the recovery itself recoverable.
