**Key differences from Git:**
- `jj` uses the term **bookmark** instead of **branch**. Bookmarks are lightweight labels on commits, similar to Git branches.
- `jj bookmark set` moves an existing bookmark (like `git branch -f`), while `jj bookmark create` creates a new one (like `git branch`).
- `jj edit` sets a commit as the working copy directly (similar to `git checkout`), but unlike Git it doesn't create a detached HEAD тАФ jj always has a mutable working copy concept.
