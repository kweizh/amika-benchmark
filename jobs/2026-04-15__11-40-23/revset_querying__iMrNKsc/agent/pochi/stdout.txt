In `jj` revset syntax: - `author("Bob")` — commits authored by Bob - `::main` — ancestors of `main` (inclusive) - `~::main` — NOT ancestors of `main` - `::@` — ancestors of the working copy (inclusive) - `&` — intersection

Written to `/home/user/project/revset.txt`:

```
author("Bob") & ~::main & ::@
```

**Breakdown of each component:**

| Component | Meaning |
|-----------|---------|
| `author("Bob")` | All commits where the author name contains "Bob" |
| `~::main` | Negation of `::main` — excludes all ancestors of `main` (and `main` itself) |
| `::@` | All ancestors of the current working copy `@` (inclusive) |
| `&` | Intersection — commits that satisfy **all** conditions |

This revset returns commits by Bob that are reachable from `@` but have **not** yet been merged into `main` — i.e., Bob's commits that exist only on the current branch above `main`.
