# Rebase a Feature Branch and Track the Rebased Bookmark

## Context

You are a platform engineer maintaining a monorepo stack for your team. Your repository has a `main` bookmark that serves as the integration branch. A colleague has been developing a new feature on a branch called `feature/add-metrics`, and the work consists of two commits stacked on top of an older base of `main`.

Since then, `main` has advanced with new infrastructure changes ÔÇö specifically, a commit that adds a `deploy.sh` script. Your task is to rebase the entire `feature/add-metrics` branch on top of the updated `main` bookmark so the feature work incorporates the latest infrastructure, and then confirm that the `feature/add-metrics` bookmark is correctly tracking the rebased tip.

## Repository State

The working directory is `/home/user/monorepo`. The repository has already been initialized with `jj git init`. The commit history looks like this:

```

Ôöé
o  feature-commit-2 ÔÇö "feat: add metrics dashboard endpoint"
Ôöé                     [feature/add-metrics bookmark]
Ôöé

Ôöé
o  old-main ÔÇö "chore: initial project scaffold"
Ôöé             [main bookmark ÔÇö OLD POSITION]
Ôöé
o  infra-update ÔÇö "chore: add deploy.sh for CI pipeline"
Ôöé                 [main bookmark ÔÇö CURRENT POSITION after update]
Ôöé

```

Wait ÔÇö more precisely, the actual topology is:

- `root()` Ôćĺ commit A (`chore: initial project scaffold`) ÔćÉ `main` was here

## Your Task

Perform the following steps in order:

### Step 1: Rebase the `feature/add-metrics` branch onto `main`

Use `jj rebase` with the `--source` (or `-s`) flag to rebase the root of the feature branch (commit C, the first feature commit) and its descendants onto the current `main`. This will replay both feature commits on top of the latest `main`.

```
jj rebase -s 'roots(main..feature/add-metrics)' -d main
```

This command says: find the root of the range between `main` and `feature/add-metrics`, and rebase that commit (along with its descendants) onto `main`.

### Step 2: Verify the rebased history

Inspect the log to confirm the feature branch is now on top of `main`:

```
jj log -r 'main::feature/add-metrics'
```

You should see two feature commits as descendants of `main`, with `main` as their ancestor.

### Step 3: Confirm the bookmark is tracking correctly

Verify that the `feature/add-metrics` bookmark has been automatically moved to the rebased tip:

```
jj bookmark list
```

You should see `feature/add-metrics` pointing to a commit whose parent is the commit pointed to by `main`.

## Expected Outcome

After completing this task:

1. **The `feature/add-metrics` bookmark** must point to a commit whose description is `feat: add metrics dashboard endpoint`.
2. **The parent of the `feature/add-metrics` tip** must be a commit whose description is `feat: scaffold metrics module`.

## Tips

- In jj, `jj rebase -s <revset> -d <destination>` rebases the given revision(s) and all their descendants onto the destination.
- The `roots(x..y)` revset finds the root(s) of the range ÔÇö i.e., commits in `x..y` that are not descendants of other commits in `x..y`.
- After a rebase, bookmarks that pointed to the rebased commits are automatically moved to the corresponding new commits.
- You do not need to manually update the bookmark after rebase; jj handles this automatically.
