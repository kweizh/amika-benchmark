## Fix Applied

**Root cause:** The heredoc delimiter `EOF` was unquoted, so the host shell expanded `$AMIKA_AGENT_CWD` before the script was sent to the sandbox. Since the variable isn't set on the host, it expanded to an empty string.

**Fix:** Quote the heredoc delimiter as `'EOF'`. When the delimiter is quoted (with single quotes), bash treats the entire heredoc body as a literal string — no variable expansion occurs on the host. The variable `$AMIKA_AGENT_CWD` is transmitted verbatim and evaluated inside the remote sandbox.

### Change in `run_agent.sh`

```diff
-amika sandbox ssh test-sandbox -- bash <<EOF > /home/user/workspace/output.log
+amika sandbox ssh test-sandbox --remote -- bash <<'EOF' > /home/user/workspace/output.log
 echo "CWD: $AMIKA_AGENT_CWD"
 EOF
```

Also added `|| true` to the `sandbox create` call so `set -e` doesn't abort when the sandbox already exists (HTTP 409).
